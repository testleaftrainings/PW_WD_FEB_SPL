1. Stabilizing Locators (Strict Mode Violation)
Problem: page.locator('button') is too generic. If a "Loading Spinner" also uses a <button> tag, Playwright detects two elements and throws a Strict Mode Violation error.

Playwright Strict Mode rule:
👉 One locator must match exactly one element.

If it finds more than one → error.

Example error:

strict mode violation: locator("button") resolved to 2 elementsYou wrote locator like this:

page.locator('button')

But webpage contains multiple buttons.

page.getByRole('button',{name:'Submit',exact:true})

Correct Solution – Semantic Locator

Instead of tag name, use Role + Accessible Name

page.getByRole('button',{name:'Submit'})

Why this works?

Browser internally creates Accessibility Tree

Example:

HTML

<button>Submit</button>
What is this?

Accessibility Tree

Role: button
Name: Submit

Playwright reads this tree.

So locator becomes very accurate.

3. Relative XPath for Dynamic Tables
Logic: When IDs are dynamic, use a unique text anchor (like a Username) to find the row, then navigate to the button within that row.

<tr>
<td>John Doe</td>
<td><button class="btn-edit">Edit</button></td>
</tr>

Method 1 – Following Sibling
//td[text()='John Doe']/following-sibling::td/button

Step-by-step:

1 find td with John Doe
2 move to next td
3 find button

4.  Soft Assertions (expect.soft)

Failure: If expect.soft() fails, the test continues to run until the end.

Reporting: The test will be marked as "Failed" in the report, showing all soft failures.

Hard Assertion Interaction: If a standard "Hard" assertion (await expect(...)) fails after a soft failure, the test stops immediately, but both the hard failure and the previous soft failures are included in the log.

5. Compound XPath (AND Logic)

Logic: Use the and keyword within square brackets to verify multiple attributes on a single element.
XPath: //input[@class='form-control' and @title='Username']

6. Backward Traversal (Preceding Axis)
Logic: The preceding axis searches for all elements located before the current element in the entire document. Using index [1] selects the closest match moving backward.

<span class="error">Invalid input</span>
<button>Save</button>

//button[text()='Save']/preceding::span[@class='error'][1]

1 find Save button
2 move backward in DOM
3 find span error
4 pick nearest one

7.Difference between slice() and splice()

Feature	slice()	splice()
Purpose	Returns a portion of an array	Adds/Removes elements in the original array
Modifies Original Array?	❌ No (non-destructive)	✅ Yes (destructive)
Return Value	Returns a new array	Returns the removed elements
Syntax	array.slice(start, end)	array.splice(start, deleteCount, item1, item2, ...)
Example	js<br>let a = [1,2,3,4,5];<br>let b = a.slice(1, 4); // [2,3,4]<br>console.log(a); // [1,2,3,4,5]<br>	js<br>
let a = [1,2,3,4,5];<br>a.splice(1, 2); // removes 2 elements from index 1<br>console.log(a); // [1,4,5]<br>

In short:
•slice() → Extracts, no change to original.
•splice() → Cuts and can insert, modifies original.\

8.How do you convert an array to a string? And string to array?

Array to String
You can use:
•	toString() or
•	join() (you can specify a separator)
let arr = ['Apple', 'Banana', 'Mango'];
let str1 = arr.toString();    // "Apple,Banana,Mango"
let str2 = arr.join(' - ');   // "Apple - Banana - Mango"


String to Array
You can use:
•	split() method
let str = "Apple,Banana,Mango";
let arr = str.split(",");   // ['Apple', 'Banana', 'Mango']
